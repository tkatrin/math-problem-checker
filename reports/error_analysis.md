# Error analysis

## Почему старые 1.000 были проблемой

Первая версия baseline обучалась через обычный `train_test_split` по строкам. В датасете несколько строк относятся к одной и той же задаче: корректный шаг, испорченный шаг, подозрительный шаг и финальный ответ. При случайном split часть шагов одной задачи могла попасть в train, а часть в test. Это создавало утечку шаблонов: модель фактически видела ту же задачу и похожие формулировки во время обучения.

Текущая версия использует `GroupShuffleSplit` по `problem_id`. Все шаги одной задачи теперь находятся либо в train, либо в test. Метрики стали ниже и реалистичнее:

| Модель | Label accuracy | Label macro-F1 | Error step F1 | Error type macro-F1 |
| --- | ---: | ---: | ---: | ---: |
| Rule-based baseline | 0.6265 | 0.5936 | 0.6545 | n/a |
| TF-IDF + numeric + SymPy LogReg | 0.9486 | 0.9638 | 0.9583 | 0.9633 |

## Где модель ошибается

На текущем synthetic benchmark основная путаница возникает между:

- `correct` и `incorrect` в коротких вычислительных шагах, где текст похож, а отличие находится только в числе;
- `calculation_error` и `missing_condition`, когда шаг содержит формулу, но ошибка на самом деле связана с выбранным знаменателем или условием;
- `none` и ошибочными типами для финальных ответов, если предыдущий контекст короткий.

## Что даёт SymPy

SymPy-признаки помогают там, где текстовые n-граммы слабы:

- `sympy_error` ловит простые арифметические ошибки;
- `check_equation_equivalence` проверяет равенства вида `left = right`;
- `check_linear_equation_solution` валидирует найденный `x` в линейном уравнении;
- `check_derivative` даёт сигнал для простых производных;
- `check_fraction_probability` ловит повторение вероятности при выборе без возвращения.

## Ограничения

Датасет всё ещё синтетический, а значит модель может переобучаться на стиль генератора. Следующие шаги:

- добавить реальные ученические решения;
- вручную проверить часть LLM/синтетической разметки;
- сделать GroupKFold по семействам задач, а не только по `problem_id`;
- обучить transformer baseline и сравнить его с TF-IDF;
- добавить отдельный holdout из задач, которых нет в генераторе.
